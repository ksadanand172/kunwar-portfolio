Deep dives

Technical highlights

Engineering problems I've solved that most Flutter engineers haven't encountered.

01
Offline-First Sync Engine
Built a queued API replay system in Hive. Mutations written locally first, replayed in-order with idempotency keys on reconnect.
Zero data loss
02
Native SDK Bridging
Wrapped Android/iOS lending SDKs using MethodChannel and EventChannel. Handles async callbacks, lifecycle events, and cross-platform error propagation for 2000+ loan flows.
2000+ loans
03
Cold Start Optimization
Cut startup 2.8s → 1.8s (35%) by profiling main() and deferring non-critical GetIt registrations post-frame. Only auth, DI, and nav bootstrap eagerly.
35% faster
04
Shorebird OTA + Feature Flags
Dart-layer hot patches with Shorebird. Critical fintech bugs fixed and rolled out in minutes, not days — no Play Store review wait. Feature flagging for safe staged rollouts.
Minutes not days

On-Device ML Attendance
Face detection + embedding matching with ML Kit entirely on-device. No server round-trip. Secure storage for facial embeddings. Works fully offline for 300+ crew.
On-device AI

Fintech Security Stack
